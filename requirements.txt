# never published for Python 3.9+ and are unavailable on PyPI for any modern
# Python/OS/CUDA combination -- `pip install -r requirements.txt` fails outright
# on any current machine before the project even runs, independent of any code
# bug. Also, torch.jit.load()'d models are generally forward-compatible with
# newer torch releases (this is the whole point of TorchScript), so pinning to
# an ancient exact version is unnecessary and actively harmful. Relaxed to a
# minimum-version floor within the 2.x series, which supports the
# torch.meshgrid(..., indexing=...) API used in utils.py and is installable on
# Python 3.9-3.12 with or without CUDA.
